What is Vulnerability Management?

Vulnerability Management is the continuous process of finding, evaluating, fixing, and monitoring security weaknesses in your systems before attackers can exploit them. Think of it like regularly checking your house for broken locks or open windows — except for your IT environment.

Why is it Important?

  • Prevents hacking and cyber attacks
  • Protects sensitive user and system data
  • Detects weaknesses early before attackers find them
  • Helps meet security standards and compliance requirements
  • Reduces cost of emergency fixes after a breach

Common Things Checked:

  • Software versions and missing patches
  • Open ports and network services
  • User access permissions and misconfigurations
  • Weak passwords and authentication settings
  • Known security flaws in applications

Key Metric: Mean Time to Remediate (MTTR) — how fast you are closing gaps after finding them. The shorter the time, the better your security posture.

What is Threat Modeling?

Threat Modeling is the process of thinking like an attacker before they attack. You look at what you are building and ask: who would want to abuse this, how would they do it, and what is the worst that could happen? The goal is to find problems during design — when fixing them is cheap — rather than after launch.

Why is it Important?

  • Finding a design flaw early costs almost nothing to fix
  • Finding it after launch — or after a breach — costs enormously
  • Helps teams build security into systems, not just add it on later
  • Improves developer thinking about how systems can be abused
  • Supports security reviews and audit requirements

The Golden Rule: Do it early. Threat modeling done during design is powerful. Done after launch, it is just documentation. The earlier you think like an attacker, the less it costs to stop them.

What is Security Auditing?

A security audit is like an inspection of your system. It checks whether security settings are correct, if there are vulnerabilities, if security policies are being followed, and if data is protected.

What is Security Assessment?

A security assessment goes deeper than an audit. It analyzes risks, possible attacks, impact, and level of risk (low, medium, or high). Not only checking the doors, but also analyzing how someone could break in and how serious the damage would be.

Why is it Important?

  • Prevents hacking and cyber attacks
  • Protects user data
  • Detects system weaknesses early
  • Improves overall system security
  • Helps meet security standards and requirements

Common Things Checked:

  • Password strength and security
  • User access control (who can access what)
  • Data encryption (is data protected?)
  • System logs (activity tracking)
  • Software updates and patches
  • Protection against attacks like SQL Injection and XSS

Vulnerability Scanning Tools

Vulnerability scanning tools are automated tools that check systems for known security flaws. They work by comparing the system against a database of known vulnerabilities, scanning for weak points in networks, servers, or applications, and generating reports with detected issues and their severity level.

How do they work?

  • The tool scans a system or network
  • It identifies services, software versions, and configurations
  • It matches them with known vulnerabilities
  • It produces a report with risks and recommendations

Why are they important?

  • Detect security weaknesses early before attackers exploit them
  • Improve system security posture
  • Reduce risk of cyber attacks
  • Help organizations follow security standards and compliance
  • Support continuous monitoring of systems

Types of Scanning Tools:

  • Network-based: Scan networks, routers, firewalls, and open ports
  • Host-based: Scan individual computers for misconfigurations and missing updates
  • Web application: Scan websites for vulnerabilities like SQL injection and XSS

What is Penetration Testing?

Penetration testing (pentesting) is an authorized simulated attack performed on a system to evaluate its security. It checks whether the system can be exploited by attackers, if there are unpatched vulnerabilities or misconfigurations, if sensitive data can be accessed without authorization, and if security defenses are working properly.

Why is it Important?

  • Identifies real vulnerabilities before attackers do
  • Tests the effectiveness of existing security measures
  • Protects sensitive user and organization data
  • Helps meet security compliance and standards
  • Reduces the risk of costly data breaches

Types of Penetration Testing:

  • Black Box: Tester has no prior knowledge of the system
  • White Box: Tester has full knowledge of the system (code, architecture)
  • Grey Box: Tester has partial knowledge of the system

Common Tools: Metasploit, Nmap, Burp Suite, Wireshark

Common Vulnerabilities and Exposures (CVE)

A CVE (Common Vulnerabilities and Exposures) is a publicly known security vulnerability assigned a unique identification number. CVEs standardize vulnerability reporting, help organizations identify threats, facilitate vulnerability management, and improve communication among security professionals.

CVE Format: CVE-2021-44228

  • CVE = Common Vulnerabilities and Exposures
  • 2021 = Year assigned
  • 44228 = Unique vulnerability number

CVE Lifecycle:

  • Vulnerability discovered
  • Report submitted
  • CVE ID assigned
  • Public disclosure
  • Security patch released

Why CVEs Matter: Without CVEs, every company might describe the same vulnerability differently. CVEs provide a common reference so everyone knows exactly which vulnerability is being discussed.

LEVEL 7: MISCONFIGURATIONS

Mission: Identify dangerous configuration mistakes.

🔑

Default Passwords

Using default credentials makes systems easy targets. Always change default passwords immediately.

🚪

Open Ports

Unnecessary open ports provide attack surfaces. Close unused ports and implement firewalls.

👤

Excessive Permissions

Over-privileged accounts increase risk. Follow principle of least privilege.

🗄️

Public Databases

Exposed databases can lead to data breaches. Never expose databases directly to the internet.

📦

Unpatched Software

Outdated software contains known vulnerabilities. Keep all systems updated with security patches.

Common Security Misconfigurations

Security misconfigurations are mistakes in how a system is set up that make it less secure and easier for attackers to exploit. Click each weakness above to learn more.

Why Does It Happen?

  • Human error
  • Lack of security knowledge
  • Failure to update systems
  • Improper security policies

Impact:

  • Unauthorized access
  • Data breaches
  • Malware infections
  • Financial and reputational damage

Prevention:

  • Change default settings
  • Regular updates and patching
  • Limit user privileges
  • Conduct security audits
